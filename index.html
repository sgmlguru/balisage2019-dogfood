So based on what Adam wrote and my understanding of it, I was able to eventually produce this

                            declare
    %rest:POST("{$body}")
    %rest:path("/paypal-transaction-complete")
function pay:listener($body) {
    let $json := fn:parse-json(util:base64-decode($body))

    return
    xmldb:store($muk:transactions-collection-uri,concat('uid-',$json?custom,'.xml'),
    <transaction tstamp='{current-dateTime()}' 
        orderID='{$json?orderID}'
        userID='{$json?custom}'/>),
    pay:all-good()
};

declare function pay:all-good(){
    (: Reply with an empty 200 response to indicate to paypal the IPN was received correctly. :)
        (
            <rest:response>
                 <http:response status="200" reason="OK"/>
            </rest:response>,
            ""
        )
};

This, of course, is not a full listener

A *listener* listens UNTIL it gets the OK and a verbatim copy of the IPN (which it sends back with an added string)

This does not repeat anything, it merely says OK and saves two parameters

Still don't fully understand this; did I mention my ignorance?

NEVERTHELESS, this gives me the order ID and the user ID (which I use to couple the transaction with the delegate)
